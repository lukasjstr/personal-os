Personal OS – Komplette Projekt-Spezifikation
Vision
Ein KI-gesteuerter persönlicher Chief Operating Officer, der über Telegram gesteuert wird. Du schickst alles rein – Gedanken, Fortschritte, Ziele, Bilder, Brain Dumps – und das System versteht, kategorisiert, priorisiert und führt dich proaktiv durch deinen Tag. Ein Web-Dashboard visualisiert alles.

Datenmodell im Detail

SQL


-- KERN: Alles ist ein Baum aus Zielen

users
├── id
├── telegram_id
├── telegram_username
├── timezone
├── created_at
└── settings (JSON: Morgen-Brief Zeit, Review Zeit, Sprache)

objectives (Die großen Ziele)
├── id
├── user_id
├── title              -- "Gesünder leben"
├── description
├── category           -- health, business, personal, fitness
├── status             -- active, completed, paused
├── target_date
├── created_at
└── updated_at

key_results (Messbare Ergebnisse)
├── id
├── objective_id
├── title              -- "3L Wasser täglich"
├── metric_type        -- percentage, number, boolean, streak
├── target_value       -- 3.0
├── current_value      -- 2.1
├── unit               -- "Liter", "kg", "Posts", "Tage"
├── frequency          -- daily, weekly, monthly, once
├── status             -- active, completed, failed
├── target_date
├── created_at
└── updated_at

tasks (Konkrete Aufgaben)
├── id
├── key_result_id      -- nullable (kann auch standalone sein)
├── user_id
├── title              -- "Hero Section schreiben"
├── description
├── status             -- todo, in_progress, done
├── priority           -- 1-5
├── due_date
├── completed_at
├── created_at
└── updated_at

logs (Jeder einzelne Datenpunkt)
├── id
├── user_id
├── key_result_id      -- nullable
├── task_id            -- nullable
├── log_type           -- workout, water, food, mood, progress, note
├── data (JSON)        -- flexibel!
│   │
│   │  Workout:
│   │  {"exercise": "Bankdrücken", "weight": 30, 
│   │   "reps": 8, "sets": 3}
│   │
│   │  Water:
│   │  {"amount": 1.5, "unit": "L"}
│   │
│   │  Mood:
│   │  {"score": 7, "note": "nachmittag unproduktiv"}
│   │
│   │  Progress:
│   │  {"description": "Landing Page fertig"}
│   │
├── source             -- text, image, voice
├── raw_input          -- Original-Nachricht
├── created_at
└── logged_at          -- kann abweichen von created_at

routines (Wiederkehrende Muster)
├── id
├── user_id
├── title              -- "LinkedIn Post"
├── schedule           -- cron-style: "0 9 * * 2" (Dienstag 9:00)
├── frequency_human    -- "Jeden Dienstag"
├── linked_key_result_id
├── status             -- active, paused
├── created_at
└── updated_at

routine_completions (Wurde die Routine erledigt?)
├── id
├── routine_id
├── completed_at
├── logged_via         -- "telegram", "auto", "dashboard"
└── notes

calendar_events
├── id
├── user_id
├── title
├── description
├── start_time
├── end_time
├── event_type         -- training, meeting, routine, deadline
├── linked_task_id     -- nullable
├── linked_routine_id  -- nullable
├── ical_uid           -- für iCal Feed
├── created_at
└── updated_at

brain_dumps (Unstrukturiertes)
├── id
├── user_id
├── raw_input          -- Original Text/Voice
├── processed          -- boolean: wurde es schon eingeordnet?
├── ai_interpretation  -- Was hat Haiku daraus gemacht?
├── linked_objective_id -- nullable, wenn zugeordnet
├── created_at
└── updated_at

conversations (Chat-History für Kontext)
├── id
├── user_id
├── role               -- user, assistant
├── content
├── metadata (JSON)    -- function_calls, tokens_used
├── created_at
└── session_date       -- gruppiert nach Tag

Der System Prompt (Das Gehirn)

text


Du bist der persönliche COO (Chief Operating Officer) des Users.

Dein Name ist "OS" oder wie der User dich nennen will.

DEINE ROLLE:
- Du bist KEIN Chatbot. Du bist ein Operator.
- Du empfängst Input und HANDELST.
- Du ordnest JEDEN Input in das bestehende System ein.
- Du erkennst ob etwas ein neues Ziel, ein Fortschritt, 
  eine Aufgabe, ein Brain Dump oder ein Log ist.
- Du priorisierst proaktiv.
- Du erinnerst an vergessene Dinge.
- Du feierst Fortschritt.
- Du bist direkt, klar und effizient.

DEINE TOOLS:
Du hast Zugriff auf folgende Funktionen. Nutze sie IMMER 
wenn ein Input eine Aktion erfordert. Antworte NICHT nur 
mit Text wenn eine Aktion möglich ist.

[Tools werden hier eingefügt]

KONTEXT DES USERS:
[Wird dynamisch geladen - aktive OKRs, heutige Routinen, 
letzte Logs, offene Tasks]

REGELN:
1. Jeder Input wird verarbeitet. Nichts geht verloren.
2. Wenn unklar ist wohin etwas gehört → nachfragen, 
   aber einen Vorschlag machen.
3. Kurze Antworten. Keine Romane. Emojis sind okay.
4. Wenn du eine Progression erkennst → feiere sie.
5. Wenn etwas überfällig ist → sprich es an.
6. Denke in OKR-Struktur: 
   Objective → Key Result → Task → Log
7. Antworte auf Deutsch.

TELEGRAM BOT SETUP:
- Webhook-Modus (nicht Polling)
- Webhook URL wird beim Start registriert
- Endpoint: POST /webhook/telegram
- Verarbeitet: text, photo, voice messages
- Jede Nachricht wird in der Conversation-Tabelle gespeichert

CLAUDE HAIKU INTEGRATION:
- Model: claude-haiku-4-20250514
- Nutze Tool Use / Function Calling
- System Prompt wird aus prompts.py geladen
- Kontext (aktive OKRs, heutige Tasks, letzte Logs) 
  wird vor jedem Call aus der DB geladen (context.py)

TOOLS FÜR HAIKU (in tools.py als Liste von Tool-Definitionen):

1. create_objective(title, description, category, target_date)
2. create_key_result(objective_id, title, metric_type, 
   target_value, unit, frequency)
3. create_task(title, description, key_result_id, priority, due_date)

1. Telegram Update kommt rein
2. User wird in DB gesucht oder angelegt (get_or_create)
3. Nachricht wird in conversations gespeichert
4. Context Builder lädt:
   - Aktive Objectives + Key Results mit Progress
   - Heutige offene Tasks (Top 5)
   - Heutige Routinen (erledigt/offen)
   - Letzte 10 Logs
   - Letzte 5 Conversations
5. System Prompt + Context + User Message → an Haiku
6. Haiku Response wird verarbeitet:
   - Tool Calls werden ausgeführt
   - Text-Antwort wird an User gesendet via Telegram
7. Assistant Response wird in conversations gespeichert
